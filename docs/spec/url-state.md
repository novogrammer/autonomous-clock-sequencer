# URL State

## URL 方針

現時点または将来使う URL パラメーター名は以下。

- `bpm`
- `beatsPerLoop`
- `stepsPerBeat`
- `kit`
- `pattern`
- `swing`

## 共通原則

- 共有対象になる state は URL に載せやすい形を優先する
- 一時的な内部状態や実行中オブジェクトは URL に載せない
- 通常の URL 更新は `history.replaceState()` を使う
- 毎操作で履歴を増やさない

## ローカル専用値

- 端末ごとの補正値や runtime 専用値は URL に載せない
- 例: `playbackOffsetMs` は端末ごとのローカル再生補正値として扱う
- `playbackOffsetMs` は共有グリッドには混ぜず、音声スケジューリングや表示確認だけに使う

## kit

- `kit` は URL の解釈ルール
- トラック数、トラック順、各トラックの意味を定義する
- UI 上の並びと `pattern` のトラック順は一致させる
- 音高を持つ `kit` では、下が低音、上が高音になる並びを基本とする
- 互換性が壊れる変更を行う場合、別 ID にする

## pattern

- `pattern` は 1 つの文字列
- 各トラックは kit で定義された順番に並ぶ
- 各トラックは `0/1` 文字列
- 区切りは `_`

例:
`pattern=1000100010001000_0000100000001000_1010101010101010`

## pattern 読み込み時

- 足りない場合は補完する
- 長すぎる場合は切り捨てる
- 不正文字は警告しつつ `0` 扱いにする
- エラーで停止しない
- 補完が必要な領域は常に `0` で埋める
- `stepsPerBeat` や `beatsPerLoop` の変更時も、既存の `1` を推測で再配置しない
